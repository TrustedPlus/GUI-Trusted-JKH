НАЧАЛО РАБОТЫ С ПРИЛОЖЕНИЕМ. ИНТЕРФЕЙС

Работа с приложением Trusted eSign GOST начинается со стартовой страницы (рис.1.1), на которой расположены кнопки перехода к мастерам приложения. Окно приложения имеет фиксированные размеры.

В верхней левой части окна расположена кнопка вызова главного меню приложения

, через которое можно выполнить переход ко всем представлениям.

При первом запуске приложения в домашней папке пользователя создается подкаталог с наименованием .Trusted. Данный подкаталог содержит файловые объекты, необходимые для корректного функционирования приложения. В частности, в подкаталоге размещаются импортированные в приложение цифровые сертификаты пользователей, ключи и списки отзыва. В файле settings.json сохраняются пользовательские настройки.

Мастер подписи/проверки - представление приложения для создания электронной подписи группы файлов и ее проверки.

Мастер шифрования/расшифрования - представление для шифрования группы файлов в адрес получателей и расшифрования принятых шифрованных файлов. Список получателей формируется на основе выбора цифровых сертификатов.

Мастер управления сертификатами - представление приложения управления сертификатами (и закрытыми ключами), которые используются для выполнения операций подписи и шифрования.


Запуск приложения на рабочем месте сразу дает возможность проверить его функциональность - в хранилище сертификатов будет присутствовать тестовый сертификат с помощью которого можно выполнить операции подписи и шифрования файлов.



СОЗДАНИЕ ЭЛЕКТРОННОЙ ПОДПИСИ

Представление мастера подписания/проверки подписи имеет три функциональных элемента: слева располагаются области выбора сертификата подписчика и настройки подписи, справа - область формирования списка файлов для выполнения операций.

Выставленные настройки сохраняются при переходе по вкладкам, а также при закрытии приложения. В виде настроек подписи передаются следующие параметры:

- Кодировка - сохранение подписи в одной из двух кодировок BASE64 или DER.

- Сохранить подпись отдельно - при установленном флажке подпись сохраняется отдельно от исходного файла.

- Добавить время подписи - при установленном флажке в подпись сохраняется время (системное) подписи.

- Директория для сохранения файла - при выбранной директории подпись сохраняется в данной директории. В противном случае подпись размещается рядом с исходным файлом.


Выбор сертификата подписчика. Для того, чтобы выполнить попись необходимо выбрать цифровой сертификат, к которому привязан закрытый ключ. Эта операция производится нажатием кнопки Выбрать сертификат подписи

В появившемся диалоговом окне отображаются сертификаты, которые могут использоваться для подписи. У отображаемых в списке сертификатов присутствует закрытый ключ.

Выбор сертификата подписчика осуществляется его выделением и нажатием на кнопку Выбрать.

Допускается смена выбранного сертификата с помощью кнопки в верхней части элемента.

Выбор подписываемых файлов. В приложении доступно создание подписи для одного или группы выбранных файлов. Выбранные файлы заносятся в правую область и представляют собой одноуровневый список. Для данного списка доступно контекстное меню в заголовке функционального элемента, состоящее из пунктов:

Выделить все - выделяются все добавленные в список файлы.

Сбросить выделение - отменяется выделение всех выбранных в списке файлов.

Удалить все из списка - список очищается. При очистке списке файлы из файловой системы не удаляются.

В списке отображается наименование файла с расширением и дата его создания. Для каждого элемента списка доступно контекстное меню, состоящее из пунктов:

Открыть файл - выполняется открытие файла через приложение, которое ассоциировано с его расширением.

Перейти к файлу - выполняется открытие каталога в котором располагается файл.

Удалить из списка - файл удаляется из текущего списка выбранных файлов для подписания. При выполнении это операции файл остается в файлововй системе в неизменном виде.

Подписание файлов. При условии выбора сертификата подписчика и подписываемых файлов в мастере становится доступной кнопка Подписать. Нажатие на эту кнопку запускает процесс подписи. Выбранные файлы подписываются по очереди. Для подписанный файлов меняется иконка, наименование, дата создания.

Если в настройках подписи не задан каталог для сохранения подписанных файлов, они сохраняются в тех же каталогах, где размещаются исходные подписываемые файлы.



ПРОВЕРКА ЭЛЕКТРОННОЙ ПОДПИСИ

Для проверки подписи достаточно выбрать проверяемые файлы - файлы с расширением .sig, которые содержат электронную подпись и нажать на кнопку Проверить. Никаких дополнительных манипуляций при проверке подписи производить не нужно.

Важно! Если при проверке отделенной от подписываемого файла подписи исходный файл не будет найден автоматически, будет предложен его выбор.

Результат проверка подписей отображается в виде сообщения.




ШИФРОВАНИЕ ФАЙЛОВ

Представление мастера шифрования/расшифрования имеет три функциональных элемента: слева располагаются области выбора сертификатов получателей и настройки шифрования, справа - область формирования списка файлов для выполнения операций.

Настройки шифрования применяются только к текущему представлению и не сохраняются при его смене. В виде настроек шифрования передаются следующие параметры:

- Кодировка - сохранение шифрованного файла в одной из двух кодировок BASE64 или DER.

- Удалить файлы после шифрования - исходные файлы, над которыми выполняется операция шифрования удаляются из файловой системы в случае успешного завершения операции.

- Архивировать перед шифрованием - файлы архивируются (ZIP) перед выполнением операции шифрования. Шифруется созданный ZIP-архив.

- Директория для сохранения файла - при выбранной директории зашифрованный файл сохраняется в данной директории. В противном случае зашифрованный файл размещается рядом с исходным файлом.

Выбор сертификатов получателей. Для того, чтобы выполнить шифрование необходимо выбрать цифровые сертификаты получателей шифрованных файлов. Выбранные на этапе шифрования получатели смогут расшифровать файлы, если у них имеется закрытый ключ. Операция выбора осуществляется нажатием кнопки Выбрать сертификаты получателей.

В появившемся диалоговом окне отображаются сертификаты, которые могут использоваться для шифрования. Допускается выбор нескольких сертификатов, так как число получателей может быть различным. Выделенные сертификаты перемещаются в правый список и по ним можно посмотреть детальную информацию.

Если список сертификатов получателей составлен, его можно подтвердить нажатием на кнопку Выбрать.

Допускается изменение списка сертификатов получателей с помощью кнопки в верхней части элемента.

Выбор шифруемых файлов. В приложении доступно шифрование одного или группы выбранных файлов. Выбранные файлы заносятся в правую область и представляют собой одноуровневый список. Для данного списка доступно контекстное меню в заголовке функционального элемента, состоящее из пунктов:

Выделить все - выделяются все добавленные в список файлы.

Сбросить выделение - отменяется выделение всех выбранных в списке файлов.

Удалить все из списка - список очищается. При очистке списке файлы из файловой системы не удаляются.

Шифрование файлов. При условии выбора сертификатов получателей и шифруемых файлов в мастере становится доступной кнопка Зашифровать. Нажатие на эту кнопку запускает процесс шифрования. Выбранные файлы шифруются по очереди, если не выбрана опция предварительной архивацмм. Для шифрованных файлов меняется иконка, наименование, дата создания.

Если в настройках подписи не задан каталог для сохранения шифрованных файлов, они сохраняются в тех же каталогах, где размещаются исходные файлы.



РАСШИФРОВАНИЕ ФАЙЛОВ

Для расшифрования достаточно выбрать файлы - файлы с расширением .enc, и нажать на кнопку Расшифровать.

Важно! Если в хранилище сертификатов не окажется сертификата с закрытым ключом который был выбран в качестве сертификата получателя при шифровании, расшифрование не будет выполнено.



УПРАВЛЕНИЕ СЕРТИФИКАТАМИ И КЛЮЧАМИ

Для управления сертификатами и ключами в приложении добавлено отдельное представление, которое связано с локальным системным хранилищем. При первом запуске приложения в хранилище уже присутствует тестовый сертификат и привязанный к нему ключ. Для того, чтобы выполнить импорт нового сертификата в хранилище можно воспользоваться контекстным меню - выбрать операцию Импортировать сертификат. В появившемся диалоговом окне нужно выбрать сертификат (поддерживааются кодировки BASE64 и DER) и сертификат помещается в хранилище. У импортированного в хранилище сертификата отсутствует привязка к закрытому ключу. Для того, чтобы выполнить импорт закрытого ключа и привязать его к выбранному сертификату нужно вызвать контекстное меню и выбрать пункт Импортировать ключ. В появившемся диалогом окне выбрать файл закрытого ключа и он будет привязан к сертификату. Важно! В текущей версии программы поддерживается импорт закрытого ключа из контейнера формата PKCS#8 без пароля.



ЛИЦЕНЗИЯ

Для установки лицензии необходимо нажать на кнопку ВВЕСТИ КЛЮЧ. В открывшемся окне введите сам лицензионный ключ или просто укажите путь к файлу лицензии.
