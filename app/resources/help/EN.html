GETTING STARTED WITH THE APP. INTERFACE

Using Trusted eSign starts with the starting page (Fig.1.1). There are navigation buttons to the wizards of the app. The application window has a fixed size.

In the upper left of the app window there is a button to call the main application menu

through which you can navigate to all views.

When you first start the application in its installation directory, a subdirectory is created, named CertStore. This subdirectory contains the file objects required for the correct functioning of the application. In particular, the digital user certificates are imported in the subdirectory.

CREATING THE DIGITAL SIGNATURE

The wizard screen for signing/signature verification has three functional elements: on the left side there is an area to choose the certificate of the subscriber and signature settings, on the right side there is an area to form the list of files to perform operations.

Selected settings may be applied only to the current choice and are not saved if changed. Settings of the signature include the following parameters:

- Encoding type - save the signature in one of two encodings, BASE64 or DER.

- Save signature as detached - when this checkbox is selected, the signature is being saved separately from the original file.

- Add time of signature - when the checkbox is selected, the signature is being saved with the information about (system) time.

- Save the file to - when the directory is selected, the signed document is saved in this directory. Otherwise, the signature is being saved next to the original file.


The choice of the user’s certificate. In order to sign the file, you must select the digital certificate that is attached to the private key. This operation is performed by clicking the button to Select the signature certificate

In the appeared dialog box the certificates are displayed which may be used for signing files. The certificates displayed have the private key attached.

The subscriber’s certificate should be selected and the Select button is to be pressed.

It is allowed to change the selected certificate using the button at the top of the element.

The choice of files to be signed. The application is available for generating a signature for a single file or a selected group of files. The selected files are entered in the right pane and represent a single-level list. This list has a context menu in the header of the functional element, consisting of items:

Select all - all listed files are selected.

Delete selection - cancels selection of all selected files added to the list.

Remove all listed files - the files from the list are removed. After removing the files from the list, they are not deleted from the file system.

The list contains the names of the files with their extensions and the dates of the signature. For each element in the list a context menu is available. It consists of the following points:

Open the file - the file is opened using the application that is associated with its extension.

Go to the file - the directory containing file is opened.

Delete the file from the list - deletes the file from the current list of files, selected for signing. When performing this operation, the file remains unchanged in the file system.

Files signing. When selecting the user’s certificate and files to be signed, the button Sign becomes available. Clicking on this button launches the signing process. The selected files are signed one by one. Signed files have different icon, name, and the date of signing.

If the directory in signature settings is not specified, the signed files are saved in the same directory where the original files are placed.



VERIFICATION OF THE ELECTRONIC SIGNATURE

To verify the signature you have to select verified files. These files have the file extension .sig, the contain an electronic signature. Click on the Verify button. No further additional manipulation to verify the signature are not needed.

Attention! When verifying the detached signature, the file has to be in the same directory.

The result of the signature verification is displayed in the message.




FILE ENCRYPTION

Representation of a encryption/decryption wizard has three functional elements: there are the recipients’ certificates and encryption settings on the left side, on the right side the user can create a list of files to perform operations.

Encryption settings are applied only to the current view and are not saved if changed. In the form of encryption include the following parameters:

- Encoding type - save the encrypted file in one of two encodings BASE64 or DER.

- Delete files after encryption - original files are deleted from the file system in case of successful completion of encryption.

- Archive before encryption - files are archived (ZIP) before performing the cryptographic operation of encryption. The process allows to create encrypted ZIP files.

- Save the file to - the signed file is saved in the specified directory. Otherwise, the signature is saved next to the original file.

Selection of the recipients’ certificates. In order to encrypt the file you must choose a digital certificate of the recipient of the encrypted file. Selected recipients can decrypt the files if they have the private key. To select the recipient the user presses the button to Select before encryption.

In the dialog box the certificates are displayed that are available to be used for encryption. You may select multiple certificates as the number of recipients may be different. Selected certificates are placed on the right side. It is possible to look through the detailed information.

If the list of recipients’ certificates is completed, it can be confirmed by pressing the Select button.

It is possible to change the list of recipients’ certificates by using the button at the top of the element.

Selection of the files to be encrypted. The application is available for encryption of one file or a selected group of files. The selected files are entered on the right side and represent a single-level list. This list has a context menu in the header of the functional element, consisting of variants:

Select all - selects all files added to the list.

Delete selection - cancels selection of all listed files.

Remove all listed files - removed files from this list are not deleted from the file system.

Encryption of files. After selection of the recipients’ certificates and encryption of files the wizard button ’to Encrypt’ becomes available. Clicking on this button launches the encryption process. The selected files are encrypted one by one, if you do not select a preliminarily archivation. Encrypted files have different icons, names, the date of creation.

If the signature settings are not specified, the files are saved in the same directory where the original files are placed.



DECRYPTION

To decrypt the file it is necessary to select the files with the extension .enc, and click on the Decrypt button.

Attention! If there is not a certificate with the private key in the certificate store which was chosen as the recipient's certificate for encryption, decryption will fail.



CERTIFICATES AND KEYS MANAGEMENT

To manage certificates and keys in the application there is a part, associated with the local system store. When you first run the application, the test certificate is already present in the repository, and it is attached to the key. In order to import the new certificate in the store you can use the context menu to select the operation “Import the certificate”. In the dialog window, choose the certificate (encodings: BASE64 or DER ) and then the certificate is being placed in storage. An imported certificate is not attached to the private key. In order to import the private key and attach it to the selected certificate, you need to call the context menu and select Import key. In the dialog box select the private key file and it will be attached to the certificate. Attention! In the current version you can import the private key from the container PKCS#8 without a password.
